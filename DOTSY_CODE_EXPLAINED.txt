OVERVIEW
================================================================================

Dotsy is a multi-provider AI CLI coding assistant built from scratch by 
Sutharson. It provides an interactive terminal interface for conversing with 
AI models while working on codebases. The tool supports multiple AI providers 
(Mistral, OpenAI, Anthropic, Google) and includes powerful features like file 
manipulation, code search, bash execution, and session management.

Current Version: 17.20.119
Author: Sutharson
Repository: https://github.com/sutharson-k/dotsy

1. ENTRY POINT (cli/entrypoint.py)
----------------------------------
Purpose: Main entry point when user runs "dotsy"

What it does:
- Parses command-line arguments (prompt, agent mode, workdir, etc.)
- Checks if current directory is trusted (security feature)
- Loads configuration from ~/.dotsy/config.toml
- Routes to either:
  * Interactive mode: Opens TUI via run_textual_ui()
  * Programmatic mode: Runs headless via run_programmatic()

Key code to explain:
```python
def main() -> None:
    args = parse_arguments()
    
    # Handle workdir change
    if args.workdir:
        os.chdir(args.workdir)
    
    # Check if interactive or programmatic
    is_interactive = args.prompt is None
    
    # Run appropriate mode
    if is_interactive:
        run_textual_ui(agent_loop, initial_prompt=args.initial_prompt)
    else:
        run_programmatic(config, prompt=args.prompt)
```

3. AGENT LOOP (core/agent_loop.py) - THE HEART OF DOTSY
--------------------------------------------------------
Purpose: Manages AI conversation and tool execution

How it works:
  User Input → AgentLoop → LLM → Tool Calls → Execute → Response → User
       ↑                                                      │
       └──────────────────────────────────────────────────────┘

Key Methods:

__init__():
- Initializes system prompt
- Creates LLM backend via BACKEND_FACTORY
- Sets up ToolManager and AgentManager
- Initializes middleware pipeline

run_turn() - Main reasoning loop:
1. Builds messages (system + conversation history)
2. Sends to LLM via backend
3. Receives response with potential tool calls
4. Executes tools (with approval if needed)
5. Sends results back to LLM
6. Repeats until task complete

_handle_tool_call():
- Validates tool permissions
- Shows approval dialog (if safety requires)
- Executes tool via ToolManager
- Returns result to LLM

Key code:
```python
async def run_turn(self, user_message: str) -> AssistantEvent:
    # Add user message to history
    self.messages.append(LLMMessage(role=Role.user, content=user_message))
    
    # Stream LLM response
    async for chunk in self.backend.complete_streaming(...):
        if chunk.message.tool_calls:
            # Execute tools
            for tool_call in chunk.message.tool_calls:
                result = await self._handle_tool_call(tool_call)
                # Send result back to LLM
```


4. LLM BACKEND (core/llm/)
---------------------------
Purpose: Abstracts different AI providers behind a common interface

Structure:
  core/llm/
  ├── backend/
  │   ├── factory.py       # Creates backend based on config
  │   ├── dotsy.py         # Dotsy/Mistral SDK integration
  │   └── generic.py       # OpenAI-compatible (OpenAI, Anthropic, Google)
  ├── format.py            # Message formatting
  ├── types.py             # Backend interface types
  └── exceptions.py        # Error handling

BackendFactory (factory.py):
```python
BACKEND_FACTORY = {
    Backend.DOTSY: DotsyBackend,
    Backend.GENERIC: GenericBackend
}

def create_backend(provider: ProviderConfig) -> BackendLike:
    backend_class = BACKEND_FACTORY[provider.backend]
    return backend_class(provider)
```

DotsyBackend (dotsy.py):
- Uses Mistral SDK directly
- complete() - Non-streaming completion
- complete_streaming() - Streaming completion
- Handles Mistral-specific features (ThinkChunk for reasoning)

GenericBackend (generic.py):
- Handles OpenAI-compatible APIs
- OpenAI (via openai SDK)
- Anthropic (via anthropic SDK)
- Google (via google-generativeai SDK)

Key interface (types.py):
```python
class BackendLike(Protocol):
    async def complete(...) -> LLMChunk: ...
    async def complete_streaming(...) -> AsyncGenerator[LLMChunk, None]: ...
```

Tool Architecture:

BaseTool (base.py):
```python
class BaseTool(ABC):
    name: str
    description: str
    parameters: dict  # JSON Schema
    
    @abstractmethod
    async def invoke(self, args: BaseModel, ctx: InvokeContext) -> ToolResult:
        """Execute the tool"""
```

ToolManager (manager.py):
- Loads built-in tools
- Loads MCP (Model Context Protocol) tools
- Validates permissions
- Executes tools with proper context

ToolPermission:
```python
class ToolPermission(StrEnum):
    ALWAYS_ASK = "always_ask"
    ALWAYS_ALLOW = "always_allow"
    NEVER = "never"
```

How tools work:
1. LLM returns tool_call with name + arguments
2. AgentLoop checks permissions
3. If approved, calls tool.invoke(args)
4. Tool executes, returns ToolResult
5. Result sent back to LLM

How it works:
```python
class MiddlewarePipeline:
    def process(self, messages: list[LLMMessage]) -> MiddlewareResult:
        for middleware in self.middlewares:
            result = middleware.process(messages)
            if result.action == MiddlewareAction.STOP:
                return result
        return MiddlewareResult.continue_()
```

   ↓

   ↓

COMMON INTERVIEW QUESTIONS & ANSWERS
================================================================================

Q: "How does Dotsy support multiple AI providers?"
A: Through the Backend abstraction in core/llm/backend/. The BackendFactory 
   creates either DotsyBackend (for Mistral) or GenericBackend (for 
   OpenAI-compatible APIs). Each implements the BackendLike protocol with 
   complete() and complete_streaming() methods.

Q: "How do tools work?"
A: Tools extend BaseTool and implement invoke(). The ToolManager loads all 
   tools, validates permissions, and executes them. When the LLM returns a 
   tool_call, the AgentLoop checks permissions, executes the tool, and sends 
   the result back.

Q: "How does the agent loop prevent infinite loops?"
A: Through middleware. TurnLimitMiddleware stops after N turns, 
   PriceLimitMiddleware stops if cost exceeds limit, and the agent can be 
   interrupted via Ctrl+C.

Q: "How does auto-scroll work?"
A: In _mount_and_scroll(), after mounting a widget, I call 
   self.call_after_refresh(self._scroll_to_bottom) which scrolls the 
   VerticalScroll container to max_scroll_y.

Q: "What's the difference between DOTSY and GENERIC backends?"
A: DOTSY uses the Mistral SDK directly and supports Mistral-specific features 
   like ThinkChunk for reasoning. GENERIC uses OpenAI-compatible APIs and 
   works with OpenAI, Anthropic, Google, and any OpenAI-compatible endpoint.
